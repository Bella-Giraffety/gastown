# Cleanup Candidate Register

This file consolidates the subsystem cleanup inventories into one canonical,
release-aware register for later epic slicing. It is evidence-only and does not
propose a combined cleanup branch.

## Normalized Fields

- `Release-aware bucket`
  - `current-path duplication`: duplication or local cleanup in the current
    supported path
  - `compatibility residue`: fallback, shim, alias, or old-layout behavior kept
    for release compatibility
  - `hybrid authority`: current behavior still overlaps with an older authority
    or lifecycle model
  - `historical docs`: stale documentation or templates with no intended happy-
    path authority
- `Current disposition`
  - `independent`: ready to slice as a standalone cleanup-only candidate
  - `sequenced`: likely valid, but should land in staged follow-up slices
  - `approval-gated`: blocked on an approval-trigger or authority-model dispute
  - `deferred`: preserve-or-migrate-first; not cleanup-safe today
